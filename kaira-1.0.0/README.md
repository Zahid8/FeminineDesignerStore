# FemDes Kaira Webstore

FemDes is a Django-backed webstore built from the Kaira fashion-store template. It keeps the original Bootstrap/Kaira storefront styling while adding database-backed products, categories, discounts, cart, checkout, orders, newsletter signup, and Django admin management.

## Current Status

The project is in active implementation. T0 through T6 are implemented, and the current agent task is `TASK-006-FIX2`, which tightens the Kaira search popup and carousel template behavior. T7 will add the demo-data seed command.

This means the site can be run locally now, but a fresh database may not show full demo catalog content until either:

- products/categories are added manually through `/admin/`, or
- the future `seed_demo_store` command from T7 is implemented and run.

## Tech Stack

- Python 3.12+
- Django 5.2.x
- SQLite for local development
- PostgreSQL for production through `DATABASE_URL`
- WhiteNoise for static files
- Bootstrap 5, jQuery, Swiper, and the existing Kaira assets

## Important Paths

- `manage.py` - Django command entrypoint.
- `femdes_site/settings.py` - environment-driven Django settings.
- `store/` - webstore app: models, admin, forms, selectors, services, views, URLs, and tests.
- `templates/` - Django templates split from the Kaira template.
- `static/store/` - Kaira CSS, JS, and images.
- `staticfiles/` - collected static output, generated by `collectstatic`.
- `media/` - uploaded product images in local development and production.
- `legacy_static/` - preserved copy of the original static template.

## Environment Variables

Create `.env` from `.env.example` for local development:

```bash
cp .env.example .env
```

Local defaults:

```env
DJANGO_SECRET_KEY=replace-with-a-long-random-secret
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=127.0.0.1,localhost
DATABASE_URL=sqlite:///db.sqlite3
DJANGO_CSRF_TRUSTED_ORIGINS=
```

Production values should look more like:

```env
DJANGO_SECRET_KEY=<long-random-secret>
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=your-domain.com,www.your-domain.com
DATABASE_URL=postgres://USER:PASSWORD@HOST:PORT/DBNAME
DJANGO_CSRF_TRUSTED_ORIGINS=https://your-domain.com,https://www.your-domain.com
```

Do not commit `.env`, `db.sqlite3`, `media/`, or `staticfiles/`.

## Run Locally With The Existing Conda Environment

The current project workflow uses the `femdes` conda environment.

```bash
conda activate femdes
python -m pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic --noinput
python manage.py createsuperuser
python manage.py runserver 127.0.0.1:8000
```

Open:

- Storefront: `http://127.0.0.1:8000/`
- Shop page: `http://127.0.0.1:8000/shop/`
- Admin panel: `http://127.0.0.1:8000/admin/`

Equivalent commands without activating the shell:

```bash
conda run -n femdes python manage.py migrate
conda run -n femdes python manage.py collectstatic --noinput
conda run -n femdes python manage.py createsuperuser
conda run -n femdes python manage.py runserver 127.0.0.1:8000
```

## Run Locally With A New Virtual Environment

Use this if the `femdes` conda environment does not exist on your machine.

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
cp .env.example .env
python manage.py migrate
python manage.py collectstatic --noinput

python manage.py runserver 127.0.0.1:8000
```

## Add Store Data

Until the T7 seed command exists, use Django admin:

1. Go to `http://127.0.0.1:8000/admin/`.
2. Log in with the superuser account.
3. Add `SiteSettings`.
4. Add active `Category` records.
5. Add active `Product` records with SKU, price, stock, and category.
6. Add `ProductImage` records if uploaded product images are needed.
7. Add `Discount` records if sale pricing should appear.

After T7 is implemented, local demo setup should become:

```bash
python manage.py seed_demo_store
```

Run it twice during verification because the seed command is required to be idempotent.

## Production Readiness Checklist

Before the site is public:

- Set `DJANGO_DEBUG=False`.
- Use a unique `DJANGO_SECRET_KEY`.
- Set `DJANGO_ALLOWED_HOSTS` to the exact production hostnames.
- Set `DJANGO_CSRF_TRUSTED_ORIGINS` to the `https://` production origins.
- Use PostgreSQL through `DATABASE_URL`.
- Run migrations on the production database.
- Run `collectstatic`.
- Create a production superuser.
- Configure persistent media storage or persistent disk for uploaded product images.
- Confirm the Kaira footer attribution remains visible unless a no-attribution license has been purchased.
- Confirm orders are manual fulfillment only; no payment gateway is implemented yet.

## Deploy To Render

Render is a straightforward way to take this Django app online. Other platforms work too, but the same concepts apply: build, migrate, collect static files, run a WSGI server, attach PostgreSQL, and configure environment variables.

### 1. Add A Production WSGI Server

The current `requirements.txt` does not include a production process server. Before deploying, add Gunicorn:

```bash
python -m pip install gunicorn
python -m pip freeze | rg "gunicorn|Django|whitenoise|dj-database-url|psycopg|Pillow|python-dotenv"
```

Then add an appropriate pinned `gunicorn` line to `requirements.txt`.

### 2. Push The Repo To GitHub

Render deploys from a Git provider.

```bash
git init
git add .
git commit -m "Prepare FemDes webstore"
git branch -M main
git remote add origin <your-github-repo-url>
git push -u origin main
```

If this directory is already inside a parent Git repo, commit from the parent repo instead of initializing a nested repository.

### 3. Create PostgreSQL

In Render:

1. Create a new PostgreSQL database.
2. Copy the internal database URL.
3. Use it as `DATABASE_URL` for the web service.

### 4. Create The Web Service

Use these settings:

- Runtime: Python
- Build command:

```bash
pip install -r requirements.txt && python manage.py collectstatic --noinput
```

- Start command:

```bash
python manage.py migrate && gunicorn femdes_site.wsgi:application
```

Set environment variables:

```env
DJANGO_SECRET_KEY=<long-random-secret>
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=<your-render-host>.onrender.com
DATABASE_URL=<render-postgres-internal-url>
DJANGO_CSRF_TRUSTED_ORIGINS=https://<your-render-host>.onrender.com
```

### 5. Create An Admin User Online

Open a Render shell and run:

```bash
python manage.py createsuperuser
```

Then visit:

```text
https://<your-render-host>.onrender.com/admin/
```

### 6. Add Data Online

Use `/admin/` to add site settings, categories, products, images, discounts, and inventory.

When T7 is implemented, you can seed demo data instead:

```bash
python manage.py seed_demo_store
```

## Attach A Custom Domain

After the Render service is healthy:

1. In Render, open the web service.
2. Add your custom domain, for example `femdes.com` and `www.femdes.com`.
3. Render will show DNS records.
4. In your domain registrar, add the required DNS records.
5. Wait for DNS propagation.
6. Update environment variables:

```env
DJANGO_ALLOWED_HOSTS=femdes.com,www.femdes.com,<your-render-host>.onrender.com
DJANGO_CSRF_TRUSTED_ORIGINS=https://femdes.com,https://www.femdes.com,https://<your-render-host>.onrender.com
```

7. Redeploy the service.
8. Confirm:

```text
https://femdes.com/
https://femdes.com/admin/
```

## Media Files In Production

Product images use Django media storage. Local uploads go under `media/`, but many hosting platforms have ephemeral filesystems. For production, use one of these:

- a persistent disk mounted to the app and configured as `MEDIA_ROOT`;
- object storage such as S3-compatible storage, added in a later task;
- manual re-upload after redeploys only for temporary testing.

Do not rely on an ephemeral deployment filesystem for real product images.

## Troubleshooting

### Static files are missing

Run:

```bash
python manage.py collectstatic --noinput
```

Confirm these files exist:

```bash
test -f static/store/style.css
test -f static/store/js/script.min.js
test -f static/store/images/main-logo.png
```

### `DisallowedHost`

Add the exact hostname to `DJANGO_ALLOWED_HOSTS`.

### CSRF errors on production forms

Add the exact `https://` origin to `DJANGO_CSRF_TRUSTED_ORIGINS`.

### Empty storefront

Add active categories and products in `/admin/`, or run `seed_demo_store` after T7 is implemented.

### Uploaded product images disappear after deploy

Configure persistent media storage. Do not use the default local `media/` directory on ephemeral hosting for real store uploads.
